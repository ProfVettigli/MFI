La Chiave Segreta: Fattori Primi

Scopri come un algoritmo apparentemente semplice — la scomposizione in fattori primi — sia il pilastro invisibile che protegge ogni tuo messaggio, ogni transazione bancaria e ogni password nel mondo digitale.

1. Il Problema: che cos'è la Scomposizione?

Ogni numero intero maggiore di 1 può essere scritto come prodotto di numeri primi — quei numeri speciali divisibili solo per 1 e per se stessi (2, 3, 5, 7, 11, 13, 17, …). Questa è la scomposizione in fattori primi, e il Teorema Fondamentale dell'Aritmetica ci garantisce che per ogni numero questa scomposizione è unica.

Definizione: La scomposizione in fattori primi di un numero intero N è l'espressione di N come prodotto di potenze di numeri primi. Ad esempio: 360 = 2³ × 3² × 5.

2. Perché è Importante? La Crittografia RSA

"Ma a che serve scomporre numeri?" — la risposta ti sorprenderà: la scomposizione in fattori primi è il fondamento della sicurezza digitale moderna. Ogni volta che invii un messaggio su WhatsApp, fai un acquisto online o accedi alla tua banca, stai usando (senza saperlo) un sistema crittografico chiamato RSA che si basa su un principio geniale:

Il Principio RSA: Moltiplicare due numeri primi enormi è facilissimo (anche per un computer vecchio). Ma partendo dal risultato, risalire ai due fattori originali è praticamente impossibile se il numero è abbastanza grande — anche con i supercomputer più potenti del mondo.

Messaggistica

WhatsApp, Signal, Telegram usano la crittografia end-to-end basata su RSA: il tuo messaggio viene "chiuso" con una chiave pubblica (il prodotto di due primi) che solo il destinatario può "aprire" con la sua chiave privata (i due fattori segreti).

Ogni transazione bancaria online (home banking, carte di credito, PayPal) usa il protocollo HTTPS che si basa su certificati digitali protetti da RSA. Senza i fattori primi, nessun hacker potrebbe fare pagamenti online.

3. Il Diagramma di Flusso dell'Algoritmo

L'algoritmo di scomposizione in fattori primi usa la strategia della divisione ripetuta: partiamo dal divisore più piccolo possibile (2) e continuiamo a dividere finché possiamo. Ecco la rappresentazione visiva completa con il diagramma di flusso:

Diagramma di Flusso

📝 Esempio: scomponiamo 180

Passo 1: 180 ÷ 2 = 90
← 2 è fattore
Passo 2:  90 ÷ 2 = 45
← ancora 2
Passo 3:  45 ÷ 2 = ?
← non divisibile! d=3
Passo 4:  45 ÷ 3 = 15
← 3 è fattore
Passo 5:  15 ÷ 3 =  5
← ancora 3
Passo 6:   5 ÷ 3 = ?
← non divisibile! d=4…5
Passo 7: d²=25 > 5
5 è primo → fattore finale!
180 = 2² × 3² × 5

  • Ovale = Inizio / Fine
  • Rettangolo = Processo (azione)
  • Rombo = Decisione (Sì/No)

4. Lo Pseudocodice

Traduciamo il workflow in pseudocodice — un linguaggio quasi-naturale che descrive la logica dell'algoritmo prima di scriverlo in un linguaggio di programmazione reale:

PSEUDOCODICE // Algoritmo: Scomposizione in Fattori Primi INIZIO LEGGI N // il numero da scomporre d ← 2 // partiamo dal più piccolo primo MENTRE d × d ≤ N ESEGUI // finché ha senso cercare MENTRE N mod d = 0 ESEGUI // se d divide N esattamente SCRIVI d // d è un fattore primo! N ← N ÷ d // continua col quoziente FINE MENTRE d ← d + 1 // prova il divisore successivo FINE MENTRE SE N > 1 ALLORA // se resta un numero > 1 SCRIVI N // è l'ultimo fattore primo FINE SE FINE

💡 Perché ci fermiamo a d² ≤ N?

È un'ottimizzazione fondamentale. Se un numero N ha un fattore primo maggiore di √N, allora l'altro fattore dev'essere più piccolo di √N — e lo avremmo già trovato! Quindi, se arriviamo a d² > N senza averlo diviso, N stesso è primo. Questa ottimizzazione riduce drasticamente i calcoli: per scomporre 1.000.000 basta testare fino a 1.000 invece che fino a 999.999.

5. L'Implementazione in Python

Ora traduciamo lo pseudocodice in un vero linguaggio di programmazione. Python è perfetto per questo — la sua sintassi è quasi identica allo pseudocodice:

PYTHON def scomponi(n): """Restituisce la lista dei fattori primi di n.""" fattori = [] d = 2 while d * d <= n: # finché d² ≤ n while n % d == 0: # se n è divisibile per d fattori.append(d) # salva d come fattore n = n // d # dividi n per d d += 1 # prossimo divisore if n > 1: fattori.append(n) # n è l'ultimo primo return fattori # Esempio di utilizzo: print(scomponi(360)) # Output: [2, 2, 2, 3, 3, 5] print(scomponi(97)) # Output: [97] → è primo!

Pseudocodice → Python

  • MENTRE...ESEGUI → while
  • SE...ALLORA → if
  • N mod d → n % d
  • N ÷ d → n // d

Simboli chiave

  • % = resto della divisione (modulo)
  • // = divisione intera (senza decimali)
  • <= = minore o uguale
  • def = definisce una funzione

7. Curiosità Finale: l'Incubo Quantistico

Abbiamo visto che tutta la sicurezza di internet si basa sul fatto che scomporre numeri enormi è praticamente impossibile per i computer classici. Ma cosa succederebbe se esistesse un tipo di computer completamente diverso, capace di violare questa regola?

⚛️

Il Computer Quantistico e l'Algoritmo di Shor

Nel 1994, il matematico Peter Shor dimostrò teoricamente che un computer quantistico — una macchina che sfrutta le leggi della meccanica quantistica (sovrapposizione e entanglement) — potrebbe scomporre in fattori primi numeri enormi in pochi minuti, laddove un computer classico impiegherebbe milioni di anni.

🔮 E quindi, siamo in pericolo?

Non ancora (nel 2026). I computer quantistici odierni sono ancora troppo instabili e "piccoli" (pochi migliaia di qubit) per attaccare le chiavi RSA reali. Ma la minaccia è così seria che governi e aziende stanno già sviluppando la crittografia post-quantistica: nuovi algoritmi di cifratura che saranno sicuri anche contro i futuri computer quantistici. È una corsa contro il tempo tra chi costruisce i computer del futuro e chi progetta le serrature del futuro!
